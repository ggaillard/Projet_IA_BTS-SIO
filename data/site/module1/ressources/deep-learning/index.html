    ## PrÃ©paration des donnÃ©es pour Machine Learning classique

Contrairement aux rÃ©seaux de neurones convolutifs (CNN), les algorithmes de ML classiques comme Random Forest ne sont pas conÃ§us pour traiter directement des images. Nous devons donc :

1. RÃ©duire la dimensionnalitÃ© des donnÃ©es (784 caractÃ©ristiques est trop Ã©levÃ©)
2. Extraire des caractÃ©ristiques pertinentes

Nous utiliserons l'Analyse en Composantes Principales (PCA) pour rÃ©duire la dimensionnalitÃ© tout en conservant l'essentiel de l'information.

    Cellule 7 (Code) - PrÃ©paration des donnÃ©es

    print("\n--- PrÃ©paration des donnÃ©es pour Random Forest ---")
print("Pour le Machine Learning classique, nous devons souvent extraire des caractÃ©ristiques manuellement.")

# RÃ©duction de dimension avec PCA pour accÃ©lÃ©rer l'entraÃ®nement
print("Application d'une rÃ©duction de dimension (PCA)...")
n_components = 50  # RÃ©duire de 784 Ã  50 caractÃ©ristiques

# SÃ©paration en ensembles d'entraÃ®nement et de test
# Utilisation d'un Ã©chantillon rÃ©duit pour accÃ©lÃ©rer la dÃ©monstration
X_sample = X[:10000]

X_train, X_test, y_train, y_test = train_test_split(X_sample, y_sample, test_size=0.2, random_state=42)

print(f"Taille de l'ensemble d'entraÃ®nement: {X_train.shape}")
print(f"Taille de l'ensemble de test: {X_test.shape}")

# CrÃ©ation d'un pipeline d'extraction de caractÃ©ristiques

    ('scaler', StandardScaler()),  # Normalisation des donnÃ©es
    ('pca', PCA(n_components=n_components))  # RÃ©duction de dimension par PCA
])

# Application aux donnÃ©es

print(f"Dimensions aprÃ¨s extraction de caractÃ©ristiques: {X_train_features.shape}")

    Cellule 8 (Markdown) - EntraÃ®nement du modÃ¨le

    ## EntraÃ®nement du modÃ¨le Random Forest

Nous allons maintenant entraÃ®ner un classificateur Random Forest sur nos donnÃ©es prÃ©traitÃ©es. Random Forest est un algorithme d'ensemble qui combine les prÃ©dictions de plusieurs arbres de dÃ©cision pour amÃ©liorer la prÃ©cision et contrÃ´ler le sur-apprentissage.

Principaux hyperparamÃ¨tres :
- **n_estimators** : Nombre d'arbres dans la forÃªt
- **max_depth** : Profondeur maximale de chaque arbre
- **min_samples_split** : Nombre minimum d'Ã©chantillons requis pour diviser un nÅud

    Cellule 9 (Code) - EntraÃ®nement du modÃ¨le

    print("\n--- EntraÃ®nement du modÃ¨le Random Forest ---")

# ParamÃ¨tres du modÃ¨le - vous pouvez les modifier
n_estimators = 100  # Nombre d'arbres
max_depth = 10      # Profondeur maximale des arbres
min_samples_split = 2  # Nombre minimum d'Ã©chantillons requis pour diviser un nÅud

# CrÃ©ation du modÃ¨le
rf_model = RandomForestClassifier(
    n_estimators=n_estimators,
    max_depth=max_depth,
    min_samples_split=min_samples_split,
    random_state=42,
    n_jobs=-1  # Utiliser tous les cÅurs disponibles
)

# Mesure du temps d'entraÃ®nement
start_time = time.time()
print("EntraÃ®nement du modÃ¨le en cours...")
rf_model.fit(X_train_features, y_train)
end_time = time.time()
training_time = end_time - start_time

print(f"Temps d'entraÃ®nement: {training_time:.2f} secondes")

    Cellule 10 (Markdown) - Ãvaluation du modÃ¨le

    ## Ãvaluation du modÃ¨le

Ãvaluons maintenant les performances de notre modÃ¨le Random Forest sur l'ensemble de test. Nous utiliserons plusieurs mÃ©triques :
